# 任务验证

## Purpose

根据当前 workspace 或 Project 的权威政策执行分层验证，将结果、候选身份和真实 wall-clock 耗时作为可复用证据返回并向用户报告。

## Consumer Obligations

consumer 必须提供可确认的任务范围、operation（`inspect`、`execute` 或 `cleanup`）、目标验证级别、候选边界、适用的 workspace 或 Project context，以及已有验证证据；无法提供稳定候选 identity 时必须明确该限制。consumer 需要复用落盘 evidence 时必须在使用完毕前保留它，并在最终摘要已捕获且没有后续 consumer 时通知 provider 可以清理 transient evidence。

## Minimum Guarantees

provider 必须从当前 scope 的 Rules、明确 Project context、OpenSpec artifacts 或项目开发/发布文档解析具体验证政策，区分 `minimal`、`affected` 和 `candidate`，执行最低充分验证并把证据绑定到当前候选；不得猜测命令、把较低级别结果升级为 Candidate，或要求 Git worktree 才能工作。provider 必须区分只核对 evidence 的 `inspect`、启动验证命令的 `execute` 和处理 evidence 生命周期的 `cleanup`，并同时支持用户直接验证意图、Agent 准备声称实现完成时的自动验证节点，以及 consumer 通过 capability binding 的调用；不得要求用户主动说出 Skill 或 capability 名称。

## Effects and Authorization

执行用户已授权任务范围内的只读或常规构建/测试命令属于验证动作；会修改外部系统、部署环境、持久业务数据、共享状态或产生其他超出项目验证政策的副作用时，必须在执行前另行取得授权。provider 可以在确认精确 cleanup reference 属于自己创建的 transient run 且 consumer 已使用完毕后删除该 run；调用方管理的稳定路径或 CI artifact 不得被擅自删除。

## Result Evidence

返回 `operation`、验证 `level`、`status`、`policySources`、`candidateIdentity`、逐项 `checks`、整体 `totalDurationMs`、`timingSource`、`slowestCheck`、`failedChecks`、`skippedChecks`、`reusable`、`taskVerificationExecuteCalls`、`candidateExecutorCalls`、可用时的 `evidenceReference`，以及落盘证据的 `evidenceRetention`、`cleanupAfter`、`cleanupStatus` 和可用时的 `cleanupReference`。`inspect` 和 `cleanup` MUST 返回两个 execute count 为 `0`；`evidenceRetention` 至少区分 `transient` 与 `caller-managed`；`timingSource` 只能是已核对 verifier summary 的 `verifier-reported` 或进程外单调时钟的 `wrapper-measured`；不得相加并行检查耗时推算整体 wall-clock。

## Decision Points

无法确认 operation、验证政策、Candidate 入口、候选 identity 或 summary 归属，验证仍在运行、必要检查失败，或验证将触发未授权副作用时必须停止、等待同一进程或返回 `incomplete`/`failed`；候选内容变化后旧 evidence 必须失效。`inspect` 或 `cleanup` 请求不得启动 verifier；无法证明 cleanup reference 属于本次 transient run、仍有 consumer、或清理失败时必须保留现场并返回 `cleanupStatus: retained`，不得扩大删除范围。

## Allowed Variations

provider 可以选择项目适用的命令发现方式、执行工具、非 Git snapshot identity、诊断格式、证据存储位置和安全清理实现，但必须保持三级语义、候选等价性、真实 wall-clock、失败状态、证据生命周期和用户披露字段兼容。`task-worktree-lifecycle` 不拥有与 checkout 无关的验证证据清理。
