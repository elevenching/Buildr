# 前端工程师角色设计候选

> Roadmap：尚未实现。本文保留未来 FE 角色 Agent 的职责设想，不是当前 Rule、Skill、Agent runtime 资产或可直接执行的角色协议。

## 候选工作前置

未来实现需要决定如何让角色获得以下上下文，而不是由本文直接触发加载：

1. 当前组织、项目、前端服务仓以及相关 `AGENTS.md`。
2. 相关 `proposal.md`、OpenSpec spec、`design.md`、设计稿、接口说明和 `tasks.md`。
3. 目标端，例如 Web/H5、小程序、App 或管理后台。
4. 代码开发所需的任务 worktree。

技术栈、构建命令和组件库应以组织级、项目级和服务仓 `AGENTS.md` 为准。

## OpenSpec 协作设想

未来前端角色可在 change 中补充：

- 影响范围：页面、入口、组件、路由、状态、接口、权限、埋点、兼容性。
- 交互口径：加载、空态、错误、禁用、确认、撤销、权限差异。
- 验证方式：构建、lint、单测、端到端测试、截图或手工验收路径。

## 实现原则候选

- 遵循项目既有目录结构、组件库、状态管理、请求封装和样式体系。
- 优先复用已有组件和业务模式；不为一次性页面过度封装。
- 完整考虑表单、列表、弹窗、筛选、分页、权限、空态、错误和加载态。
- UI 文案以业务动作和状态为准。
- 关注 XSS、敏感信息暴露、越权入口和不可信输入展示。
- 适配目标端常见屏幕尺寸，破坏性操作提供明确确认机制。

## 验证与交付设想

未来角色应优先运行项目既有的 lint、测试和构建命令，并在无法运行时说明原因和替代校验。若任务存在 OpenSpec change，应同步更新对应 `tasks.md` 勾选状态。
