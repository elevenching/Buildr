# Mermaid 图表语法注意事项

编写Mermaid图表时，请遵循以下规则避免解析错误：

## 1. 禁止在节点和连接线文本中使用引号

* 不要在节点或连接线上使用任何引号(`"`, `'`, `“”`, `\"`)  
* 否则会引起`got 'STR'`类型的错误
* 示例：
  * ✗ `A[运营进入 "代理人管理" 列表页]`
  * ✓ `A[运营进入 代理人管理 列表页]`

## 2. 避免在节点文本中使用圆括号`()`

* Mermaid对圆括号非常敏感，即使在`[]`或`{}`内也会导致`got 'PS'`错误
* 示例：
  * ✗ `M[填写方案名称/描述(可选)]` 
  * ✓ `M[填写方案名称/描述]`

## 3. 避免在节点文本中使用方括号`[]`

* 即使在其他节点形状内使用方括号也可能导致`got 'SQS'`错误
* 示例：
  * ✗ `C{打开[代理人名称]-业务方案管理抽屉}`
  * ✓ `C{打开 代理人名称 - 业务方案管理 抽屉}`

## 4. Subgraph标题包含特殊字符时使用引号

* 当subgraph标题包含`>`, `(`, `)`等特殊字符时，要用双引号将整个标题括起来
* 示例：
  * ✗ `subgraph 管理流程 (运营后台 - 业财管理 > 资金账户管理)`
  * ✓ `subgraph "管理流程 (运营后台 - 业财管理 > 资金账户管理)"`

## 5. 保持简洁

* 使用简洁的节点和连接线文本，减少特殊符号
* 这样可以显著降低出错概率，减少调试时间
