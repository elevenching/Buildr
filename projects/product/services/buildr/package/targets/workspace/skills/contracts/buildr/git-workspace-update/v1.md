# Git Workspace 更新

## Purpose

在执行 Buildr sync 前，安全更新由 Git 管理的 workspace。

## Consumer Obligations

consumer 必须明确 workspace root、当前 Agent、预期 upstream，以及更新成功后是否继续执行 workspace sync。

## Minimum Guarantees

provider 必须检查分支、upstream、分叉、冲突和本地改动；不得静默 stash、覆盖、rebase，也不得自行选择缺失的 upstream。

## Effects and Authorization

用户请求的安全更新可以改变本地 checkout。改写历史、丢弃改动、强制操作和远端写入需要单独授权。

## Result Evidence

返回分支、upstream、更新前后的 commit、更新方式、尚未解决的决策点，以及 `treeChanged`。

## Decision Points

遇到无法保留的本地改动、需要选择策略的分叉、冲突、缺失 upstream 或不受支持的仓库状态时，必须停止。

## Allowed Variations

在保留停止条件和结果证据的前提下，provider 可以实现组织自定义的 pull、fetch-and-integrate、评审或分支策略。
