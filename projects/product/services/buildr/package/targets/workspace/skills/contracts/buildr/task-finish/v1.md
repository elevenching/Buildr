# 任务收尾

## Purpose

安全编排已验证任务的完整收尾，但不接管 provider 专属的 Git 或 worktree 策略。

## Consumer Obligations

consumer 必须明确 task/change、请求的收尾范围、验证证据、目标分支、远端，以及所有明确排除的动作。

## Minimum Guarantees

provider 必须解析 required capabilities，在写入前披露 selected providers 和影响；任一阶段失败时停止，保持验证证据与 tree 等价性，并记录 optional 阶段的降级。

## Effects and Authorization

当前轮次的收尾授权覆盖已披露的常规归档、任务提交、集成、推送和安全本地清理。强制推送、删除远端分支、丢弃改动、改写共享历史和语义冲突决策仍明确排除。

## Result Evidence

返回各阶段结果、provider identities、验证证据、归档的 change（如有）、提交/集成/推送结果、清理结果、跳过的 optional 阶段和剩余风险。

## Decision Points

遇到 required capability 被阻断、范围或目标不明确、验证不可信、语义冲突、检查失败、远端写入被拒绝或无法安全清理时，必须停止。

## Allowed Variations

provider 可以调整编排细节和 optional 审查的处理方式，但必须把 Git 集成和 worktree 生命周期策略交给各自的 selected provider。
