# Git 任务集成

## Purpose

将已验证任务提交并集成到约定的目标分支，同时提供 Git 影响与候选内容转换的可审计证据。

## Consumer Obligations

consumer 必须提供任务范围、目标分支、远端、输入 candidate identity 及其验证证据、明确排除的改动，以及本次调用获准产生的影响。

## Minimum Guarantees

provider 必须保留无关工作，对比 Git 操作前后的 candidate content identity，并在出现语义冲突时停止；默认只推送已集成的目标分支，且不得因为任务分支存在、已提交或已合入而创建或推送远端任务分支；不得执行项目 Candidate 验证、决定验证 evidence 是否有效，或把传输成功当作验证成功。

## Effects and Authorization

只允许执行事先披露的提交、集成和推送影响。远端任务分支只有在用户当前轮次明确要求且已披露分支、远端、目的和待推送提交时才可推送。强制推送、改写共享历史、删除远端分支、丢弃改动和解决语义冲突，都需要单独的明确授权。

## Result Evidence

返回纳入和排除的改动、commit identities、目标 ref 的前后状态、目标分支远端结果、任务分支未推送状态或明确授权的远端结果、输入与最终 candidate content identity、tree 等价性信号，以及 `treeChanged`。tree 等价性信号只描述 Git 操作效果；验证 evidence 的有效性、复用或重跑由 task-verification provider 或其 consumer 决定。

## Decision Points

无法确定任务范围、目标、远端、授权、输入 candidate identity、冲突处理方式或最终 content identity 等价性时，必须停止。

## Allowed Variations

只要在写入前完成披露且符合上述保证，provider 可以自行决定 rebase、merge、fast-forward、提交信息、评审和托管策略。
