# Git 单项操作

## Purpose

安全执行一个明确的 Git 操作意图，不将其扩展为完整任务收尾。

## Consumer Obligations

consumer 必须明确仓库、请求的操作、受影响的 refs，以及用户当前授予的权限。

## Minimum Guarantees

provider 必须检查相关 Git 状态、保留无关改动，并在存在语义歧义时停止；不得静默扩大用户请求的操作范围。

## Effects and Authorization

本地只读检查属于隐含授权。提交、改写历史、合并、推送、删除分支、修改远端状态或丢弃改动，必须取得与该具体影响相匹配的授权。

## Result Evidence

返回仓库、执行的操作、受影响的 refs、适用时的结果 commit 或 tree identity、远端影响，以及 `treeChanged` 布尔值。

## Decision Points

遇到冲突、不安全的 dirty tree、目标 refs 不明确、远端写入被拒绝，或现有授权未覆盖所需影响时，必须停止。

## Allowed Variations

在满足最低保证并提供规定结果证据的前提下，provider 可以自行决定命令、提交策略、分支命名、托管约定和协作策略。
